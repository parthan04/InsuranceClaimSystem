# SECURITY SIDE 

✅ 🔐 BACKEND SECURITY CHECKLIST (CRITICAL LAYER)

Backend = Trust Boundary / Security Enforcement

✅ 1️⃣ Authentication & Authorization

✔ Strong password policy
✔ Password hashing (bcrypt / argon2 / PBKDF2)
✔ No plain text passwords EVER
✔ Role-based access control (RBAC)
✔ Verify permissions on EVERY request
✔ Protect admin routes
✔ Session expiration / timeout
✔ Re-authentication for sensitive actions

❌ Never trust frontend roles / flags

✅ 2️⃣ CSRF Protection

✔ CSRF tokens
✔ SameSite cookies
✔ Validate Origin / Referer
✔ Framework protection enabled (Django / Spring / etc.)

✅ 3️⃣ Input Validation (VERY IMPORTANT 🔥)

✔ Validate ALL inputs server-side
✔ Type checking
✔ Length limits
✔ Whitelisting > Blacklisting
✔ Reject unexpected fields
✔ Sanitize dangerous characters when needed

Protects against:

✔ SQL Injection
✔ XSS
✔ Command Injection
✔ Path Traversal

✅ 4️⃣ SQL Injection Prevention

✔ Use ORM / Prepared Statements
✔ Parameterized queries ONLY
✔ No raw SQL concatenation

✔ Validate numeric inputs strictly

✅ 5️⃣ File Upload Security

✔ Validate file type (MIME + Extension)
✔ Rename uploaded files
✔ Store outside web root
✔ File size limits
✔ Block executables/scripts
✔ Scan files if sensitive system

✅ 6️⃣ Path Traversal Protection

✔ Never trust file paths from users
✔ Normalize paths
✔ Restrict directories
✔ Use safe file handlers

✅ 7️⃣ Session & Cookie Security

✔ Secure cookies (Secure flag)
✔ HttpOnly cookies
✔ SameSite cookies
✔ Session rotation after login
✔ Session expiration
✔ Logout invalidation

✔ HTTPS ONLY

✅ 8️⃣ Brute Force Protection

✔ Rate limiting
✔ Login throttling
✔ Account lockout
✔ CAPTCHA (optional)
✔ IP monitoring

✅ 9️⃣ Business Logic Protection

✔ Validate workflow order
✔ Prevent privilege escalation
✔ Prevent price / amount tampering
✔ Prevent duplicate transactions
✔ Verify ownership of resources

Most dangerous vulnerabilities live here 🔥

✅ 🔟 Error Handling

✔ No stack traces in production
✔ No DB errors exposed
✔ Generic error messages

❌ Never leak internal details

✅ 1️⃣1️⃣ Security Headers

✔ CSP
✔ HSTS
✔ X-Frame-Options
✔ X-Content-Type-Options
✔ Referrer-Policy

Usually via backend / reverse proxy.

✅ 🖥 FRONTEND SECURITY CHECKLIST (SAFETY LAYER)

Frontend = Attack Surface Reduction / Safe Rendering

✅ 1️⃣ XSS Prevention 🔥

✔ Avoid innerHTML
✔ Use safe DOM APIs
✔ Escape output
✔ Sanitize dynamic content
✔ Use framework auto-escaping (React / Vue / etc.)

✔ Treat ALL API data as untrusted

✅ 2️⃣ Safe Token Handling

✔ Prefer HttpOnly cookies
✔ Avoid storing tokens in localStorage (if sensitive)
✔ Never expose secrets
✔ Protect against token leakage

✅ 3️⃣ Input Handling (UX Layer)

✔ Validate inputs for UX
✔ Length limits
✔ Format checks
✔ But NEVER rely on this alone

✅ 4️⃣ Secure API Usage

✔ Always HTTPS
✔ No sensitive data in URLs
✔ Avoid exposing internal IDs unnecessarily
✔ Proper headers

✅ 5️⃣ Sensitive Data Protection

✔ Never expose:

❌ API keys
❌ Secrets
❌ Internal configs
❌ Debug data

✔ Remove console logs in production

✅ 6️⃣ CSP Compatibility

✔ Avoid inline scripts
✔ Avoid unsafe eval
✔ Helps block XSS injection

✅ 7️⃣ Clickjacking Awareness

✔ Respect X-Frame-Options
✔ Avoid unsafe embedding

✅ 8️⃣ Dependency Security

✔ Audit third-party libraries
✔ Avoid outdated packages
✔ Remove unused libraries

Frontend supply-chain attacks are real 🔥

✅ 9️⃣ Proper Error Display

✔ No technical errors shown to users
✔ No internal API responses leaked
